## 2. Прогноз рентабельности (ML)

Источник данных: помесячные метрики после синхронизации с Google Таблицей.  
CatBoost: отложенная валидация на последних до **6** точках (`n_val = min(6, len(work)//3)`), цель — сдвинутая на месяц вперёд рентабельность (`target = profitability.shift(-1)`).

| Показатель | Модель ARIMA | CatBoost | Ансамбль | Примечание |
|------------|--------------|----------|----------|------------|
| Горизонт прогноза, мес. | **≤ 12** | **≤ 12** | **≤ 12** | В коде: `horizon = min(horizon_months, 12)`; все режимы используют один горизонт |
| Длина истории обучения, мес. | Ряд до **36** последних мес. (`MAX_TRAIN_MONTHS`); для подгонки ARIMA нужно **≥ 4** точек | Тот же `df` после признаков; полноценный прогноз CatBoost при **≥ 12** мес. в исходном ряду (`MIN_HISTORY_FOR_FULL`) | Те же входы, что у ARIMA и CatBoost | Если число месяцев **меньше** `FORECAST_MIN_MONTHS_HISTORY`, для всего ответа включается **naive** (в `.env.example` задано **12**; в `app/config.py` по умолчанию **6** — переопределяется env) |
| MAE (валидация), п.п. | — | **не зафиксировано в отчётном прогоне** | — | Ошибка считается на `eval_set` внутри `CatBoostRegressor.fit`; в JSON ответа API поле **`metrics` пустое** |
| MAPE / sMAPE, % | — | **не зафиксировано** | — | При необходимости — offline по факту на отложенном окне или доработка логирования в `forecast_ml.py` |
| Наблюдение | Порядок (p,d,q) по **AIC**, стационарность **ADF** | **400** итераций, depth **6**, lr **0,05**, early stopping **25** | **0,5·ARIMA + 0,5·CatBoost**, если обе ветки дали прогноз; иначе доступная модель | Винзоризация ряда (п. 5–95), клип прогноза **[-100; 100]** п.п., постобработка сезонности `_apply_seasonal_variation`, при сбое — naive |

**Как получить MAE/MAPE в цифрах:** временно вывести в лог MAE на `eval_set` после `model.fit` в `forecast_ml.py` или сравнить `y_pred` и факт в notebook/скрипте по историческим месяцам.
